# crosshud v4

бесплатный open-source прицел-оверлей для windows.

crosshud дает гибкую настройку прицела, пиксельный редактор, шаблоны, rgb-эффекты, поведение под пкм, горячие клавиши и работу из системного трея. версия 4 — это финальная переработка ветки 3.1: новый qml-интерфейс, более модульное ядро оверлея, встроенное обновление и нормальная диагностика для поддержки.

## основные возможности

- обычный прицел: размер, толщина, зазор, цвет и прозрачность.
- дополнительные элементы: центральная точка, цвет точки, размер точки, обводка и толщина обводки.
- rgb-эффекты: радужный режим и динамический цвет.
- пиксельный редактор: сетка 101x101, zoom 6x-12x, перемещение средней кнопкой мыши, undo/redo через кнопки и `ctrl+z`, `ctrl+y`, `ctrl+shift+z`.
- шаблоны: сохранение, загрузка и удаление пользовательских пиксельных прицелов.
- поведение: скрытие или переключение прицела по пкм, горячая клавиша включения, автозапуск, скрытый запуск.
- системный трей: быстрое открытие окна и корректный выход.
- обновления: проверка github releases, баннер в приложении, скачивание установщика, прогресс, запуск установки и перезапуск.
- диагностика: просмотр логов, uid пользователя, сведения о системе и zip-отчет для отправки в telegram `@petyablatnoy`.

## установка

1. откройте страницу `releases`.
2. скачайте `crosshud_by_petyablatnoy_setup.exe`.
3. запустите установщик от имени администратора, если windows запросит права.
4. после установки запустите crosshud из меню пуск или с ярлыка.

установщик ставит приложение в `program files`, добавляет нужный сертификат для подписи и проверяет подпись после установки. путь установки заблокирован намеренно: для uiaccess и стабильной работы поверх окон windows приложение должно находиться в системной папке.

## использование

- вкладка прицела отвечает за форму, цвет, прозрачность, точку, обводку и rgb.
- вкладка пиксельного редактора нужна для ручного рисования прицела по клеткам.
- вкладка системы содержит разрешение, горячие клавиши, поведение пкм, логи, обновления и диагностику.
- кнопка сохранения активна только когда настройки реально отличаются от сохраненных.
- если настройки не сохранены, выход из окна или трея покажет предупреждение.

## поддержка

если у пользователя ошибка:

1. открыть `система`.
2. в блоке `сведения` нажать `zip`.
3. отправить созданный архив в telegram `@petyablatnoy`.

архив содержит `system_info.txt`, `system_info.json` и до 5 последних файлов логов. автоматической отправки нет: пользователь сам решает, отправлять отчет или нет.

## технический стек

- python 3.
- pyside6.
- qml / qt quick controls 2 для интерфейса.
- pillow для отрисовки прицела в bgra-буфер.
- winapi через `ctypes` для layered window, topmost-окна, хуков мыши/клавиатуры и мониторинга foreground-окна.
- pyinstaller для сборки.
- nsis для установщика.

## архитектура

основные модули:

- `main_file.py` — точка входа, cli-аргументы, single-instance, логирование и запуск приложения.
- `crosshud_app.py` — главный контроллер: qt app, tray, окно, overlay, обновления и корректное завершение.
- `qml_ui.py` — мост между qml и python: настройки, шаблоны, логи, обновления, диагностика.
- `qml/` — интерфейс на qml: страницы прицела, пиксельного редактора, системы и общие компоненты.
- `overlay_manager.py` — тонкий координатор оверлея.
- `overlay_window.py` — создание и обновление прозрачного win32 layered window.
- `input_hooks.py` — горячие клавиши, пкм и foreground hook.
- `win32_api.py` — низкоуровневые winapi-структуры и вызовы.
- `crosshair_renderer.py` — чистый рендер прицела.
- `settings_manager.py` — настройки, валидация, миграция, шаблоны и dirty-state.
- `diagnostics.py` — uid, сведения о системе и support-архив.
- `app_metadata.py` — имя, версия и github-метаданные.

## сборка

требования:

- python 3.
- windows.
- nsis.
- powershell.

команда:

```powershell
python build.py
```

скрипт создает build-окружение, устанавливает зависимости из `requirements-build.txt`, собирает exe через pyinstaller, подписывает приложение, собирает nsis-установщик и подписывает установщик.

## локальная разработка

установка зависимостей:

```powershell
python -m pip install -r requirements.txt
```

запуск:

```powershell
python main_file.py
```

полезные проверки:

```powershell
python -m unittest discover -s tests -v
python -m compileall -q .
```

## публикация релиза

правильная схема публикации:

1. исходники публикуются в репозиторий обычными файлами и папками через git.
2. `sourcecode.zip` не загружается в корень репозитория.
3. установщик `crosshud_by_petyablatnoy_setup.exe` прикладывается только к github release.
4. архивы `source code (zip)` и `source code (tar.gz)` github создает сам автоматически для каждого релиза.

если в репозитории уже лежит старый `sourcecode.zip`, его нужно удалить из репозитория отдельным коммитом или заменить ветку актуальным деревом исходников перед публикацией v4.
