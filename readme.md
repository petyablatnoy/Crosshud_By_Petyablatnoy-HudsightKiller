# crosshud v4

бесплатный открытый прицел оверлей для windows.

crosshud помогает настроить удобный прицел поверх игры или другого окна: изменить форму, цвет, прозрачность, добавить точку, обводку, ргб эффекты или нарисовать полностью свой пиксельный прицел.

### зачем нужен crosshud

| задача                                     | что дает crosshud                                                    |
| ------------------------------------------ | -------------------------------------------------------------------- |
| в игре нет удобной настройки прицела       | можно использовать внешний прицел поверх окна                        |
| нужен один привычный прицел в разных играх | настройки сохраняются и применяются одинаково                        |
| хочется точной настройки                   | доступны размер, толщина, зазор, цвет, прозрачность, точка и обводка |
| нужен свой дизайн                          | есть пиксельный редактор и шаблоны                                   |
| нужен бесплатный аналог платных приложений | проект открыт и распространяется бесплатно                           |

### чем crosshud отличается от hudsight

hudsight - сильный коммерческий инструмент с платной лицензией и несколькими режимами отрисовки. crosshud не пытается копировать все подряд: его преимущество в другом - он бесплатный, открытый, понятный и сфокусированный на настройке прицела без лишней привязки к платной экосистеме.

| пункт               | crosshud                                                     | hudsight                                          |
| ------------------- | ------------------------------------------------------------ | ------------------------------------------------- |
| цена                | бесплатно                                                    | платная лицензия                                  |
| исходный код        | открыт, можно проверить и форкнуть                           | закрытый коммерческий продукт                     |
| привязка к магазину | не требует стим для работы                                   | есть стим версия и лицензирование                 |
| прозрачность        | можно увидеть, как хранятся настройки, логи и диагностика    | внутренняя логика скрыта                          |
| диагностика         | отчеты с логами и сведениями о системе для поддержки         | поддержка зависит от внешнего канала разработчика |
| пиксельный редактор | встроенный редактор 101x101 с отменой, возвратом и шаблонами | есть редактор и генератор                         |
| цель проекта        | легкий открытый инструмент для кастомного прицела            | коммерческий продукт с широкой экосистемой        |

если нужна максимальная совместимость с разными полноэкранными режимами, гейм баром, воркшопом и готовой коммерческой экосистемой, hudsight может быть удобнее. если нужен бесплатный, открытый и проверяемый инструмент без покупки лицензии, crosshud дает главное: гибкую настройку прицела, пиксельный редактор, шаблоны, обновления и поддержку через диагностический отчет.

### возможности

| раздел              | возможности                                                     |
| ------------------- | --------------------------------------------------------------- |
| основной прицел     | размер, толщина, зазор, цвет, прозрачность                      |
| элементы            | центральная точка, цвет точки, размер точки, обводка            |
| эффекты             | радужный режим и динамический цвет                              |
| пиксельный редактор | сетка 101x101, масштаб, отмена, возврат, шаблоны                |
| поведение           | скрытие или переключение по правой кнопке мыши, горячая клавиша |
| система             | автозапуск, запуск в трей, проверка обновлений, логи            |
| поддержка           | uid, сведения о системе, zip отчет с логами                     |

### установка

1. откройте страницу релизов.
2. скачайте `crosshud_by_petyablatnoy_setup.exe`.
3. запустите установщик.
4. после установки откройте crosshud через ярлык или меню пуск.

при установке windows может запросить права администратора. это нормально: приложению нужно корректно установиться в системную папку, чтобы прицел стабильно работал поверх окон.

### как пользоваться

1. откройте crosshud.
2. включите прицел на первой вкладке.
3. настройте размер, толщину, зазор, цвет и прозрачность. сохраните настройки.

### пиксельный редактор

| действие         | управление                        |
| ---------------- | --------------------------------- |
| рисовать         | лкм                               |
| стирать          | пкм                               |
| перемещать холст | скм                               |
| изменить масштаб | `ctrl + колесико`                 |
| отменить         | `ctrl + z`                        |
| вернуть          | `ctrl + y` или `ctrl + shift + z` |

пиксельный прицел можно сохранить как шаблон и загрузить позже.

### обновления

если доступна новая версия, в приложении появится уведомление. из него можно:

- обновиться автоматически;
- открыть страницу релиза;
- скрыть уведомление и обновиться позже.

### поддержка

если что-то работает неправильно:

1. откройте вкладку `система`.
2. в блоке `сведения` нажмите `zip`.
3. опишите проблему и отправьте созданный архив в telegram `@petyablatnoy`.

отчет содержит сведения о системе и последние логи приложения.

## для разработчиков

### стек

| часть          | технология                          |
| -------------- | ----------------------------------- |
| приложение     | python 3                            |
| интерфейс      | pyside6 + qml / qt quick controls 2 |
| рендер прицела | pillow                              |
| оверлей        | winapi layered окно через `ctypes`  |
| хуки           | winapi хуки мыши и клавиатуры       |
| сборка         | pyinstaller                         |
| установщик     | nsis                                |

### структура проекта

| путь                    | назначение                                                                |
| ----------------------- | ------------------------------------------------------------------------- |
| `main_file.py`          | точка входа, аргументы запуска, защита от второго экземпляра, логирование |
| `crosshud_app.py`       | главный контроллер приложения, трей, запуск интерфейса и оверлея          |
| `qml_ui.py`             | связующий слой между qml и python                                         |
| `qml/`                  | интерфейс приложения                                                      |
| `crosshair_renderer.py` | отрисовка прицела в bgra буфер                                            |
| `overlay_manager.py`    | координация окна оверлея, хуков и рендера                                 |
| `overlay_window.py`     | win32 layered окно                                                        |
| `input_hooks.py`        | пкм, горячие клавиши, хук активного окна                                  |
| `win32_api.py`          | winapi структуры и функции                                                |
| `settings_manager.py`   | настройки, валидация, шаблоны, состояние несохраненных изменений          |
| `diagnostics.py`        | uid, системные сведения, архив для поддержки                              |
| `app_metadata.py`       | версия, имя приложения, github метаданные                                 |
| `tests/`                | модульные тесты                                                           |

### локальный запуск

```powershell
python -m pip install -r requirements.txt
python main_file.py
```

полезные аргументы:

| аргумент     | назначение                  |
| ------------ | --------------------------- |
| `--minimize` | запуск свернутым            |
| `--enable`   | включить прицел при запуске |
| `--debug`    | подробное логирование       |
| `--version`  | показать версию             |

### проверки

```powershell
python -m unittest discover -s tests -v
python -m compileall -q .
```

### сборка

для сборки нужны windows, python, powershell и nsis.

```powershell
python build.py
```

скрипт создает сборочное окружение, ставит зависимости из `requirements-build.txt`, собирает приложение через pyinstaller, подписывает exe, собирает nsis-установщик и подписывает установщик.

## безопасность и открытый код

проект открыт, поэтому исходный код можно проверить. установщик и приложение подписываются, чтобы windows корректно разрешал работу оверлея.
