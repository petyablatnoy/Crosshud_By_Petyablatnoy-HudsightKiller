# changelog

## 4.0.0 - 2026-07-04

изменения составлены относительно опубликованной версии 3.1.

### для пользователей

#### главное

- новый интерфейс на qml/qt quick controls 2.
- полностью обновлен внешний вид приложения.
- добавлена отдельная боковая панель с разделами прицела, пиксельного редактора и системы.
- боковая панель отделена от остального интерфейса и аккуратно центрирует кнопки.
- добавлена сплошная тонкая рамка вокруг окна.
- улучшены всплывающие уведомления: они плавно появляются и плавно исчезают.
- обновлена иконка приложения.

#### прицел

- сохранены основные настройки: размер, толщина, зазор, цвет и прозрачность.
- сохранены центральная точка, обводка, радужный режим и динамический цвет.
- пользовательское разрешение теперь вводится аккуратнее, значения центрированы в полях.
- настройки считаются измененными только если они реально отличаются от сохраненных.
- если вернуть ползунок на прежнее значение, программа не просит сохранить настройки.

#### пиксельный редактор

- масштаб изменен на диапазон 6x-12x.
- добавлены кнопки отмены и возврата действий.
- добавлены горячие клавиши `ctrl+z`, `ctrl+y`, `ctrl+shift+z`.
- центральный пиксель теперь находится по центру области редактора при открытии.
- перемещение холста средней кнопкой мыши стало плавнее.
- исправлен сильный лаг при рисовании.
- пиксели отправляются в приложение один раз в конце мазка, а не на каждое движение мыши.

#### выход и трей

- выход через трей теперь корректно показывает окно, если есть несохраненные настройки.
- при выходе с несохраненными настройками появляется понятное предупреждение.
- исправлены подвисания при завершении через кнопку внутри программы.
- исправлены предупреждения qml в консоли при завершении.

#### обновления

- добавлен баннер обновления внутри приложения.
- системное уведомление теперь открывает приложение, а не запускает обновление напрямую.
- в баннере есть кнопки обновления, страницы релиза и отложенного обновления.
- кнопка обновления скачивает установщик, показывает прогресс, запускает установку и перезапускает приложение.
- кнопка отложенного обновления скрывает баннер с анимацией.
- проверка обновлений запускается после подготовки трея.

#### логи и поддержка

- добавлены полезные логи запуска, сохранения настроек, обновлений и завершения.
- логи обновляются в разделе системы только когда окно открыто и не свернуто.
- добавлен uid пользователя для обращений в поддержку.
- добавлены сведения о системе: windows, процессор, память, видеокарта, драйвер, монитор, dpi, настройки прицела.
- добавлен zip-отчет для телеграм `@petyablatnoy`.
- отчет содержит `system_info.txt`, `system_info.json` и до 5 последних файлов логов.

### для разработчиков

#### архитектура

- удалены старые части интерфейса на виджетах из версии 3.1.
- версия и github метаданные вынесены в `app_metadata.py`.
- оверлей разделен на логические модули:
  - `win32_api.py`;
  - `overlay_window.py`;
  - `input_hooks.py`;
  - `overlay_manager.py`.
- `overlay_manager.py` стал тонким координатором, а не крупным файлом со всей логикой winapi.
- рендер прицела вынесен в `crosshair_renderer.py`.
- системная диагностика вынесена в `diagnostics.py`.

#### qml и связь с python

- интерфейс загружается через `qml_ui.py`.
- добавлен связующий слой между qml и python для настроек, логов, шаблонов, обновлений и диагностики.
- исправлены qml предупреждения по `Shortcut` и обработчикам сигналов.
- добавлен отдельный сигнал для диагностики, чтобы системные сведения не пересчитывались при каждом изменении прицела.
- тяжелые системные запросы диагностики кэшируются за запуск.

#### настройки

- `settings_manager.py` теперь валидирует значения настроек.
- неизвестные или поврежденные настройки отклоняются или заменяются безопасными значениями.
- состояние несохраненных изменений основано на сравнении текущего состояния с сохраненным снимком.
- сохранение пиксельного прицела отделено от обычного сохранения настроек.

#### обновления

- проверка обновлений использует последний релиз репозитория.
- поиск установщика не привязан к номеру версии.
- установщик выбирается среди exe файлов релиза с `setup` в имени.
- вспомогательный скрипт обновления скачивает установщик во временную папку, запускает его и удаляет временный файл.

#### сборка и установка

- зависимости запуска закреплены в `requirements.txt`.
- зависимости сборки подключают `requirements.txt` и добавляют `pyinstaller`.
- `build.py` генерирует `version_info.txt` и `installer_version.nsh` из `app_metadata.py`.
- установщик собирается через nsis.
- exe и установщик подписываются.
- установка закреплена за `program files`, чтобы сохранить корректную работу uiaccess.
- удаление ограничено ожидаемыми путями установки.
