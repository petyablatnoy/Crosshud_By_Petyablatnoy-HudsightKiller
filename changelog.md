# changelog

## 4.0.0 - 2026-07-04

финальная версия 4.0.

### главное

- интерфейс полностью переведен на qml / qt quick controls 2.
- удалены старые хвосты widget-интерфейса из версии 3.1.
- версия и github-метаданные вынесены в `app_metadata.py`.
- оверлей разделен на логические модули: `win32_api.py`, `overlay_window.py`, `input_hooks.py` и тонкий `overlay_manager.py`.
- добавлен guided updater: системное уведомление открывает приложение, баннер показывает новую версию, кнопка обновления скачивает установщик, показывает прогресс, запускает установку и перезапускает приложение.
- добавлен support-раздел: uid, системная информация, технический блок сведений и zip-отчет с логами для telegram `@petyablatnoy`.

### интерфейс

- собран новый минималистичный qml-интерфейс с отдельной боковой панелью.
- выровнены кнопки в sidebar и добавлена цельная граница окна.
- исправлены размеры и позиционирование текста в заголовке окна.
- значения пользовательского разрешения центрированы в полях ввода.
- всплывающие предупреждения теперь плавно появляются и плавно исчезают.
- блок системной информации сделан в техническом key/value-стиле.

### пиксельный редактор

- диапазон zoom изменен на 6x-12x.
- добавлены undo/redo-кнопки и горячие клавиши `ctrl+z`, `ctrl+y`, `ctrl+shift+z`.
- центральный пиксель теперь центрируется при открытии редактора.
- исправлено дерганое перемещение холста средней кнопкой мыши.
- исправлен лаг при рисовании: редактор обновляет canvas локально во время мазка и отправляет пиксели в python один раз в конце изменения.

### настройки и выход

- dirty-state теперь сравнивает текущие настройки с сохраненным снимком, а не просто отмечает любое движение ползунка как изменение.
- если пользователь вернул ползунок на старое значение, настройки не считаются измененными.
- выход через трей теперь показывает окно и предупреждение при несохраненных настройках.
- исправлены qml-предупреждения при завершении приложения.
- завершение через кнопку внутри приложения больше не подвешивает курсор и не уничтожает qml-объект во время активного signal handler.

### логи и диагностика

- добавлены короткие полезные lifecycle-логи без лишнего мусора.
- логи автоматически обновляются в разделе системы только когда окно открыто и не свернуто.
- support-отчет включает `system_info.txt`, `system_info.json` и до 5 последних лог-файлов.
- диагностические системные запросы кэшируются за запуск и не мешают редактору.

### обновления

- проверка обновлений стартует после инициализации tray.
- исправлено отсутствие баннера обновления внутри приложения.
- кнопка `релиз` открывает github latest.
- кнопка `позже` скрывает баннер с анимацией.
- поиск установщика не привязан к конкретной версии: используется latest release репозитория и exe-asset с `setup` в имени.

### сборка и установка

- установщик собирается через nsis и использует метаданные версии из одного места.
- установщик подписывается после сборки.
- установка закреплена за `program files`, чтобы сохранить корректную работу uiaccess.
- удаление безопасно ограничено ожидаемыми путями установки.

## 3.1 - 2026-01-20

историческая опубликованная версия 3.1.

### по релизу 3.1

- переработана прозрачность оверлея.
- улучшена геометрия размера и зазора прицела.
- оптимизирован запуск и инициализация оверлея.
- область отрисовки стала фиксированной.
- добавлена проверка обновлений.
- улучшены всплывающие уведомления.
- прозрачность стала применяться ко всему прицелу, включая точку и обводку.

### техническое состояние 3.1

- интерфейс был на pyside6 widgets.
- основная ui-логика жила в `ui_components.py` и `qt_widgets.py`.
- пиксельный редактор был отдельным widget-модулем `pixel_editor.py`.
- winapi, overlay window, render loop, хуки мыши и клавиатуры были сосредоточены в крупном `overlay_manager.py`.

## 3.0

- крупная переработка после v2.
- переход на pyside6.
- добавлен пиксельный редактор.
- добавлены шаблоны, обводка, центральная точка, rgb-режим, трей и uiaccess-сборка.
